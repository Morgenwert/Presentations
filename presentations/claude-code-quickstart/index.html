First Steps

Getting Started: Essential Commands

Three commands you'll use regularly to manage your Claude Code sessions:

🚀
/init

Initialize a new project. Analyzes structure and creates a CLAUDE.md file.

🔃
/clear

Reset the conversation. Start fresh when switching to a different task.

📦
/compact

Summarize the conversation to free up context window space.

What /init does:

1
Analyzes your project structure
Scans files, folders, and dependencies
↓
2
Creates a CLAUDE.md file
Project-specific instructions for Claude
↓
3
Sets up the foundation
Claude now understands your codebase

💡 Tip

Use /compact proactively during long sessions to avoid running out of context. Use /clear when you're done with one task and starting something unrelated.

Gitignored

Personal Preferences (All Projects)

  • Your preferred coding style
  • Custom agents you use everywhere
  • Default language preferences

Team Conventions (Shared via Git)

  • Code style rules (TypeScript, ESLint, etc.)
  • Architecture guidelines
  • Team conventions
  • Important project notes

Personal Notes (Not Shared)

  • Local file paths
  • Personal experiments
  • Notes you don't want to share
Features

Key Features

📋 Plan Mode

When to use: Before implementing complex features or making architecture decisions.

How it works:

  1. Claude explores your codebase
  2. Creates a structured plan
  3. You review and approve
  4. Claude implements

💡 Trigger

Claude often enters Plan Mode automatically, or ask: "Let's plan this before implementing."

🧠 Think Modes

When to use: For debugging difficult problems or understanding complex code.

Command Depth Use Case
think Normal Standard analysis
think more Higher Complex bugs
think harder Deep Architecture problems
ultrathink Maximum Critical decisions

💾 Checkpoints

Claude automatically creates snapshots before making changes.

To undo changes:

command 
/checkpoint restore

Or simply say: "Undo the last changes"

Integrations

MCP Servers - External Integrations

MCP (Model Context Protocol) lets Claude communicate with external services.

~/.claude.json 
{
  "mcpServers": {
    "filesystem": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-filesystem", "/path/to/folder"]
    }
  }
}

Popular MCP Servers

📁
server-filesystem

File access

🐙
server-github

GitHub integration

💬
server-slack

Slack messages

🗃
server-postgres

Database access

Tips

Best Practices

✔

Do

  • Be specific about what you want
  • Mention file paths when relevant
  • Break complex tasks into steps
  • Review changes before committing
✖

Don't

  • Give vague instructions
  • Assume Claude knows your context
  • Ask for everything at once
  • Blindly accept all changes

For Saving Context

Situation Solution
Context filling up during a task Use /compact to summarize and free space
Switching to a different task Use /clear to start fresh
Research tasks Use subagents
Complex planning Use Plan Mode
